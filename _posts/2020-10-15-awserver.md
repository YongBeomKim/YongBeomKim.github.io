# **AWS INSTANCE SETTING**

앞선 내용에서 진행한 내용을 요약하면 다음과 같습니다.
1. AWS 가입절차
2. AWS Console 로그인 및 EC2 인스턴스 생성
3. **사용자 pem** 키파일을 활용한 접속

여기까지 작업하면 인스턴스의 생성 및 실행은 완료 됩니다. 이제부터 필요한 내용을 요약하면 다음과 같습니다.
1. **root** 계정의 활성화
2. **pem** 키파일 없이 로그인

## **Root 계정의 활성화**

다음의 내용은 **[EC2 root 계정 활성화](https://goddaehee.tistory.com/193)** 를 간단하게 요약하였습니다.

1. **pem** 로 서버에 접속 합니다.
2. `$sudo passwd root` 로 root 의 비밀번호를 설정 합니다.
3. `$sudo vi /etc/ssh/sshd_config` 에서 **PermitRootLogin yes** 변경 후 저장 합니다.
4. `$sudo mkdir /root/.ssh` 폴더를 생성 합니다.
5. `$sudo cp /home/ubuntu/.ssh/authorized_keys /root/.ssh` EC2 유저의 인증키를 root 에게 복사 합니다.
6. `$sudo systemctl restart sshd` ssh 를 재실행 합니다.
7. `$ssh -i 'C:\키페어경로\키페어.pem' root@접속IP` 로 접속을 확인 합니다.

## **pem 인증키 파일없이 별도 사용자 로그인 활성화**

EC2 유저 및 root 는 pem 키파일을 활용하는 방법을 추천합니다. 반면 **[추가 사용자](https://ithub.tistory.com/215)** 를 활용하면 별도의 인증키 없이도 접속 설정이 가능 합니다.

1. 최초에는 .pem 파일을 사용하여 EC2에 접속합니다.
2. 새로운 유저를 생성합니다. 
 - `$ sudo useradd -s /bin/bash -m -d /home/USERNAME -g root USERNAME`
3. 유저 비밀번호를 설정합니다.
 - `$ sudo passwd USERNAME`
4. 패스워드 입력
5. sudoers 파일 권한 변경 
 - `$ sudo chmod u+w /etc/sudoers`
6. sudoers 파일 열고, username 추가 합니다.
 - `$ sudo vi /etc/sudoers` 
 - `USERNAME ALL=(ALL:ALL) ALL`
7. sshd_config 파일에 PasswordAuthentication 설정 변경
 - `$ sudo vi /etc/ssh/sshd_config`
 - `PasswordAuthentication yes`
8. ssh를 재시작합니다.
 - `$ sudo service ssh restart`
